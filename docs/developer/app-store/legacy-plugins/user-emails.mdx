This plugin is deprecated!

If you plan on building a new integration with Saleor, we recommend using the [SMTP](../apps/smtp/overview) app instead.
:::

`UserEmails` plugin is responsible for sending emails to shop customers.

After being called by [`PluginManager`](developer/extending/plugins/overview.mdx), the plugin will fill the template using event payload and send a message via SMTP.

## Configuration

If the email configuration of the plugin on the dashboard side will be empty, the plugin will try to use variables defined in settings.py file:

- `settings.USER_EMAIL_HOST`
- `settings.USER_EMAIL_PORT`
- `settings.USER_EMAIL_HOST_USER`
- `settings.USER_EMAIL_HOST_PASSWORD`
- `settings.USER_EMAIL_USE_TLS`
- `settings.USER_EMAIL_USE_SSL`

### Note

Alternatively those variables can be provided via [`USER_EMAIL_URL`](../../../setup/configuration.mdx#user_email_url) environment variable.
Plugin configuration values will override the environment variable values.

## Available events

- `account_confirmation`
- `account_password_reset`
- `account_change_email_request`
- `account_change_email_confirm`
- `account_delete`
- `account_set_customer_password`
- `invoice_ready`
- `order_confirmation`
- `order_confirmed`
- `order_fulfillment_confirmation`
- `order_fulfillment_update`
- `order_payment_confirmation`
- `order_canceled`
- `order_refund_confirmation`
- `send_gift_card`

## Templates

`UserEmails` plugin uses [handlebars](https://handlebarsjs.com/guide/) as a template language.
You can use [Saleor's default templates](https://github.com/saleor/saleor-email-templates) as a good start for building your emails.
Prepared emails can be uploaded via the plugin section on the Dashboard side.

## Helper functions

### format_address

Applies formatting to the address.

Arguments:

- **address** (json) - Address instance in JSON format.
- **include_phone** (boolean) - Controls if the formatted address will contain a phone number. Default: True
- **inline** (boolean) - Controls if the address should be formatted inline. Default: False
- **latin** (boolean) - Controls if Unicode characters (for example Cyrillic or kanji) should be formatted as their Latin counterparts. [Learn more](https://github.com/mirumee/google-i18n-address#address-latinization). Default: False

Usage:

```
{{format_address order.shipping_address}}
```

### price

Applies formatting to the price.

Arguments:

- **net_amount** (decimal) - Net price.
- **gross_amount** (decimal) - Gross price.
- **currency** (string) - Price currency.
- **display_gross** (boolean) - Controls if gross price should used, instead of net price.

Usage:

```
{{price order.subtotal_net_amount order.subtotal_gross_amount order.currency display_gross=order.display_gross_prices}}
```

### format_datetime

Converts datetime iso string format to requested format.

Arguments:

- **date** (date) - Date you want to format.
- **date_format** (string) - Date format. By default, it will be `"%d-%m-%Y"`. Available formats can be found [here](https://docs.python.org/3/library/datetime.html#strftime-and-strptime-format-codes).

Usage:

```
{{format_datetime order.created "%b. %d, %Y"}}
```

### get_product_image_thumbnail

Returns URL to the requested size of image.

Arguments:

- **size** (integer) - Image size in pixels.
- **image** (Image) - Image instance.

Usage:

```
{{get_product_image_thumbnail 60 product.first_image}}
```

### compare

Compares two values. Returns true/false value.

Arguments:

- **val1** (any) - First value.
- **compare_operator** (string) - Available operators: `==`,`!=`,`<`,`<=`,`>=`,`>`.
- **val2** (any) - Second value.

Usage:

```
{{#if (compare order.id "==" "1")}}
    First order!
{{/if}}
```
