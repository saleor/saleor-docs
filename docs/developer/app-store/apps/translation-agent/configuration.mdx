## Pre-requisites

- Create an account at [OpenRouter](https://openrouter.ai) and generate an API key.

## Installation

Go to the Apps section in your Saleor Dashboard, find Translation Agent, and install it. Once installed, open it from the Translations menu.

## Configuration

Setup is split across three pages.

### API key and model

On the LLM config page, paste your OpenRouter API key and choose a model. The Verify button runs a small test request to confirm the key and model work together before you save.

### Languages

On the Languages page, pick your source language and the target languages you want to translate into. Both are chosen from the languages your shop already has, so there are no codes to type by hand.

### Translation settings

The Translation settings page covers three things:

- **Brand context:** a short company voice, target audience, formality, brand terms to keep verbatim, and terms to never use. This is added to every translation prompt.
- **Glossary:** term mappings the model must follow, scoped per target language.
- **Approval workflow:** turn on per-language review. When it is on, translations for that language are saved as drafts instead of going live.

## Running a translation

Open the Translate page, pick a target language, and start the run. Progress and a per-product summary appear as it works. Use the force option to re-translate products that already have a translation.

## Approval workflow

When approval is enabled for a language, a run writes drafts rather than publishing. Review them on the Approvals page: filter by language, edit a translation inline, then approve or reject. Approved drafts are published to the catalog. Descriptions render as formatted rich text in the review dialog, with a visual editor for editing the translation.

## Troubleshooting

- **The Verify button fails.** Check that the OpenRouter API key is correct and that your chosen model is available on your OpenRouter account.
- **A run translates nothing.** Incremental runs skip products that already have a translation in the target language. Use the force option to re-translate them.
- **A translated description reverts to the source.** The app validates the structure of formatted descriptions and falls back to the source when a translation does not match, which keeps broken markup off the storefront.
