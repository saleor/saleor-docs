## NOTIFY

`NOTIFY` is raised in order to trigger a new notification to the user.
Each webhook payload has `notify_event` field which defines the type of the notification event and payload.

### Customer account has been created

<details><summary>Expand example</summary>

```json
{
  "notify_event": "account_confirmation",
  "payload": {
    "user": {
      "id": 68,
      "email": "test@example.com1",
      "first_name": "",
      "last_name": "",
      "is_staff": false,
      "is_active": false,
      "private_metadata": {},
      "metadata": {},
      "language_code": "en"
    },
    "recipient_email": "test@example.com1",
    "token": "ajcv09-f3f7880d4a5e33657fe12f9cc372a23f",
    "confirm_url": "https://dev.saleor.cloud/account-confirm?email=test%40example.com1&token=ajcv09-f3f7880d4a5e33657fe12f9cc372a23f",
    "domain": "example.com",
    "site_name": "Saleor e-commerce"
  }
}
```

</details>
