No description

```graphql
type OrderReturnProductsInput {
  orderLines: [OrderReturnLineInput!]
  fulfillmentLines: [OrderReturnFulfillmentLineInput!]
  paymentsToRefund: [PaymentToRefundInput]
  amountToRefund: PositiveDecimal
  includeShippingCosts: Boolean
  refund: Boolean
}
```

### Fields

#### `orderLines` ([`[OrderReturnLineInput!]`](../../../developer/api-reference/inputs/order-return-line-input))

List of unfulfilled lines to return.

#### `fulfillmentLines` ([`[OrderReturnFulfillmentLineInput!]`](../../../developer/api-reference/inputs/order-return-fulfillment-line-input))

List of fulfilled lines to return.

#### `paymentsToRefund` ([[`PaymentToRefundInput`](../../../developer/api-reference/inputs/payment-to-refund-input)])

New in Saleor 3.0. A list of payments to be refunded when the value is provided manually.

#### `amountToRefund` ([`PositiveDecimal`](../../../developer/api-reference/scalars/positive-decimal))

The total amount of refund when the value is provided manually.

DEPRECATED: this field will be removed in Saleor 4.0. Use paymentsToRefund instead.

#### `includeShippingCosts` ([`Boolean`](../../../developer/api-reference/scalars/boolean))

If true, Saleor will refund shipping costs. If amountToRefund is provided includeShippingCosts will be ignored. If amount is provided with every payment in the paymentsToRefund, then includeShippingCosts will be ignored.

#### `refund` ([`Boolean`](../../../developer/api-reference/scalars/boolean))

If true, Saleor will call refund action for all lines.
