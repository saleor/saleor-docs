Sets the customer as the owner of the checkout.

Requires one of the following permissions: AUTHENTICATED_APP, AUTHENTICATED_USER.

```graphql
checkoutCustomerAttach(
  checkoutId: ID
  customerId: ID
  id: ID
  token: UUID
): CheckoutCustomerAttach
```

### Arguments

#### `checkoutId` ([`ID`](../../../developer/api-reference/scalars/id))

The ID of the checkout.

DEPRECATED: this field will be removed in Saleor 4.0. Use `id` instead.

#### `customerId` ([`ID`](../../../developer/api-reference/scalars/id))

ID of customer to attach to checkout. Can be used to attach customer to checkout by staff or app. Requires IMPERSONATE_USER permission.

#### `id` ([`ID`](../../../developer/api-reference/scalars/id))

The checkout's ID.

Added in Saleor 3.4.

#### `token` ([`UUID`](../../../developer/api-reference/scalars/uuid))

Checkout token.

DEPRECATED: this field will be removed in Saleor 4.0. Use `id` instead.

### Type

#### [`CheckoutCustomerAttach`](../../../developer/api-reference/objects/checkout-customer-attach)

Sets the customer as the owner of the checkout.

Requires one of the following permissions: AUTHENTICATED_APP, AUTHENTICATED_USER.
