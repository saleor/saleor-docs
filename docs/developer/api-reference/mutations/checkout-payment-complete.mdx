title: checkoutPaymentComplete
---

Complete an individual payment as part of the checkout. This mutation does not create the order and any webhooks related to this payment will neither create the order.
In case an additional confirmation step such as 3D secure is required confirmationNeeded flag will be set to True and no authorization will be held until the payment is confirmed with the second call of this mutation.

```graphql
checkoutPaymentComplete(
  paymentData: JSONString
  paymentId: ID!
  redirectUrl: String
  token: UUID!
): CheckoutPaymentComplete

```

### Arguments

#### `paymentData` ([`JSONString`](../../../developer/api-reference/scalars/json-string))

Client-side generated data that is required to finalize the payment.

#### `paymentId` ([`ID`](../../../developer/api-reference/scalars/id))

The ID of a payment.

#### `redirectUrl` ([`String!`](../../../developer/api-reference/scalars/string))

URL of a view where users should be redirected to see the order details. URL in RFC 1808 format.

#### `token` ([`UUID`](../../../developer/api-reference/scalars/uuid))

Checkout token.

### Type

#### [`CheckoutPaymentComplete`](../../../developer/api-reference/objects/checkout-payment-complete)

Complete an individual payment as part of the checkout.
