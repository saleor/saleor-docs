### Possible types

#### [`App`](/docs/developer/api-reference/objects/app)

Represents app data.

#### [`Address`](/docs/developer/api-reference/objects/address)

Represents user address data.

#### [`User`](/docs/developer/api-reference/objects/user)

Represents user data.

#### [`Group`](/docs/developer/api-reference/objects/group)

Represents permission group data.

#### [`ProductVariant`](/docs/developer/api-reference/objects/product-variant)

Represents a version of a product such as different size or color.

#### [`Product`](/docs/developer/api-reference/objects/product)

Represents an individual item for sale in the storefront.

#### [`ProductType`](/docs/developer/api-reference/objects/product-type)

Represents a type of product. It defines what attributes are available to products of this type.

#### [`Collection`](/docs/developer/api-reference/objects/collection)

Represents a collection of products.

#### [`Category`](/docs/developer/api-reference/objects/category)

Represents a single category of products. Categories allow to organize products in a tree-hierarchies which can be used for navigation in the storefront.

#### [`ProductMedia`](/docs/developer/api-reference/objects/product-media)

Represents a product media.

#### [`ProductImage`](/docs/developer/api-reference/objects/product-image)

Represents a product image.

#### [`PageType`](/docs/developer/api-reference/objects/page-type)

Represents a type of page. It defines what attributes are available to pages of this type.
