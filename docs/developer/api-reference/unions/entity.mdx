\_Entity union as defined by Federation spec.

```graphql
union _Entity =
    App
  | Address
  | User
  | Group
  | ProductVariant
  | Product
  | ProductType
  | Collection
  | Category
  | ProductMedia
  | PageType
```

### Possible types

#### [`App`](../../../developer/api-reference/objects/app)

Represents app data.

#### [`Address`](../../../developer/api-reference/objects/address)

Represents user address data.

#### [`User`](../../../developer/api-reference/objects/user)

Represents user data.

#### [`Group`](../../../developer/api-reference/objects/group)

Represents permission group data.

#### [`ProductVariant`](../../../developer/api-reference/objects/product-variant)

Represents a version of a product such as different size or color.

#### [`Product`](../../../developer/api-reference/objects/product)

Represents an individual item for sale in the storefront.

#### [`ProductType`](../../../developer/api-reference/objects/product-type)

Represents a type of product. It defines what attributes are available to products of this type.

#### [`Collection`](../../../developer/api-reference/objects/collection)

Represents a collection of products.

#### [`Category`](../../../developer/api-reference/objects/category)

Represents a single category of products. Categories allow to organize products in a tree-hierarchies which can be used for navigation in the storefront.

#### [`ProductMedia`](../../../developer/api-reference/objects/product-media)

Represents a product media.

#### [`PageType`](../../../developer/api-reference/objects/page-type)

Represents a type of page. It defines what attributes are available to pages of this type.
