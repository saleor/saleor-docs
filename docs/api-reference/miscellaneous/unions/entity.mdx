\_Entity union as defined by Federation spec.

```graphql
union _Entity =
  | App
  | PageType
  | Address
  | User
  | Group
  | ProductVariant
  | Product
  | ProductType
  | ProductMedia
  | Category
  | Collection
  | Order
```

### Possible types

#### [`App`](/api-reference/apps/objects/app.mdx)

Represents app data.

#### [`PageType`](/api-reference/pages/objects/page-type.mdx)

Represents a type of page. It defines what attributes are available to pages of this type.

#### [`Address`](/api-reference/users/objects/address.mdx)

Represents user address data.

#### [`User`](/api-reference/users/objects/user.mdx)

Represents user data.

#### [`Group`](/api-reference/users/objects/group.mdx)

Represents permission group data.

#### [`ProductVariant`](/api-reference/products/objects/product-variant.mdx)

Represents a version of a product such as different size or color.

#### [`Product`](/api-reference/products/objects/product.mdx)

Represents an individual item for sale in the storefront.

#### [`ProductType`](/api-reference/products/objects/product-type.mdx)

Represents a type of product. It defines what attributes are available to products of this type.

#### [`ProductMedia`](/api-reference/products/objects/product-media.mdx)

Represents a product media.

#### [`Category`](/api-reference/products/objects/category.mdx)

Represents a single category of products. Categories allow to organize products in a tree-hierarchies which can be used for navigation in the storefront.

#### [`Collection`](/api-reference/products/objects/collection.mdx)

Represents a collection of products.

#### [`Order`](/api-reference/orders/objects/order.mdx)

Represents an order in the shop.
