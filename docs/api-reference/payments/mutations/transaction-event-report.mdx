Report the event for the transaction.

<Permissions
  permissions={
    "OWNER and HANDLE_PAYMENTS for apps,HANDLE_PAYMENTS for staff users"
  }
  text={"Requires the following permissions"}
/>
Staff user cannot update a transaction that is owned by the app.

Triggers the following webhook events:

- TRANSACTION_ITEM_METADATA_UPDATED (async): Optionally called when transaction's metadata was updated.
- CHECKOUT_FULLY_PAID (async): Optionally called when the checkout charge status changed to `FULL` or `OVERCHARGED`.
- ORDER_UPDATED (async): Optionally called when the transaction is related to the order and the order was updated.

### Arguments

#### [<code>amount</code>](#)<Bullet />[<code>PositiveDecimal</code>](/api-reference/miscellaneous/scalars/positive-decimal.mdx)

The amount of the event to report.

Required for all `REQUEST`, `SUCCESS`, `ACTION_REQUIRED`, and `ADJUSTMENT` events. For other events, the amount will be calculated based on the previous events with the same pspReference. If not possible to calculate, the mutation will return an error.

#### [<code>availableActions</code>](#)<Bullet />[<code>[TransactionActionEnum!]</code>](/api-reference/payments/enums/transaction-action-enum.mdx)

List of all possible actions for the transaction

#### [<code>externalUrl</code>](#)<Bullet />[<code>String</code>](/api-reference/miscellaneous/scalars/string.mdx)

The url that will allow to redirect user to payment provider page with event details.

#### [<code>id</code>](#)<Bullet />[<code>ID</code>](/api-reference/miscellaneous/scalars/id.mdx)

The ID of the transaction. One of field id or token is required.

#### [<code>message</code>](#)<Bullet />[<code>String</code>](/api-reference/miscellaneous/scalars/string.mdx)

The message related to the event. The maximum length is 512 characters; any text exceeding this limit will be truncated.

#### [<code>paymentMethodDetails</code>](#)<Bullet />[<code>PaymentMethodDetailsInput</code>](/api-reference/miscellaneous/inputs/payment-method-details-input.mdx)

Details of the payment method used for the transaction.

<Badge
  text="Added in Saleor 3.22"
  class="badge badge--secondary margin-bottom--sm"
/>

#### [<code>pspReference</code>](#)<Bullet />[<code>String!</code>](/api-reference/miscellaneous/scalars/string.mdx)

PSP Reference of the event to report.

#### [<code>time</code>](#)<Bullet />[<code>DateTime</code>](/api-reference/miscellaneous/scalars/date-time.mdx)

The time of the event to report. If not provide, the current time will be used.

#### [<code>token</code>](#)<Bullet />[<code>UUID</code>](/api-reference/miscellaneous/scalars/uuid.mdx)

The token of the transaction. One of field id or token is required.

#### [<code>transactionMetadata</code>](#)<Bullet />[<code>[MetadataInput!]</code>](/api-reference/miscellaneous/inputs/metadata-input.mdx)

Fields required to update the transaction metadata. Can be read by any API client authorized to read the object it's attached to.

Warning: never store sensitive information, including financial data such as credit card details.

#### [<code>transactionPrivateMetadata</code>](#)<Bullet />[<code>[MetadataInput!]</code>](/api-reference/miscellaneous/inputs/metadata-input.mdx)

Fields required to update the transaction private metadata.

Requires permissions to modify and to read the metadata of the object it's attached to.

Warning: never store sensitive information, including financial data such as credit card details.

#### [<code>type</code>](#)<Bullet />[<code>TransactionEventTypeEnum!</code>](/api-reference/payments/enums/transaction-event-type-enum.mdx)

Current status of the event to report.
