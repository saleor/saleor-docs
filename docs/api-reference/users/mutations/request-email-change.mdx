Request email change of the logged in user.

<Permissions
  permissions={"AUTHENTICATED_USER"}
  text={"Requires one of the following permissions"}
/>

Triggers the following webhook events:

- NOTIFY_USER (async): A notification for account email change.
- ACCOUNT_CHANGE_EMAIL_REQUESTED (async): An account email change was requested.

```graphql
requestEmailChange(
  channel: String
  newEmail: String!
  password: String!
  redirectUrl: String!
): RequestEmailChange
```

### Arguments

#### [<code>channel</code>](#channel)<Bullet />[<code>String</code>](/api-reference/miscellaneous/scalars/string.mdx) \{#channel\}

Slug of a channel which will be used to notify users. Optional when only one channel exists.

#### [<code>newEmail</code>](#new-email)<Bullet />[<code>String!</code>](/api-reference/miscellaneous/scalars/string.mdx) \{#new-email\}

New user email.

#### [<code>password</code>](#password)<Bullet />[<code>String!</code>](/api-reference/miscellaneous/scalars/string.mdx) \{#password\}

User password.

#### [<code>redirectUrl</code>](#redirect-url)<Bullet />[<code>String!</code>](/api-reference/miscellaneous/scalars/string.mdx) \{#redirect-url\}

URL of a view where users should be redirected to update the email address. URL in RFC 1808 format.
