Request email change of the logged in user.

Requires one of the following permissions: AUTHENTICATED_USER.

Triggers the following webhook events:

- NOTIFY_USER (async): A notification for account email change.
- ACCOUNT_CHANGE_EMAIL_REQUESTED (async): An account email change was requested.

```graphql
requestEmailChange(
  channel: String
  newEmail: String!
  password: String!
  redirectUrl: String!
): RequestEmailChange
```

### Arguments

#### [`channel`](#)<Bullet />[`String`](/api-reference/miscellaneous/scalars/string.mdx)

Slug of a channel which will be used to notify users. Optional when only one channel exists.

#### [`newEmail`](#)<Bullet />[`String!`](/api-reference/miscellaneous/scalars/string.mdx)

New user email.

#### [`password`](#)<Bullet />[`String!`](/api-reference/miscellaneous/scalars/string.mdx)

User password.

#### [`redirectUrl`](#)<Bullet />[`String!`](/api-reference/miscellaneous/scalars/string.mdx)

URL of a view where users should be redirected to update the email address. URL in RFC 1808 format.
